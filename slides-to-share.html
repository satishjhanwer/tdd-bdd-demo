Coding Standards and Guidelines

  • Improves Code Readability and Maintainability

Naming Conventions

Using descriptive, consistent names for variables, functions, and classes so that their purpose is immediately clear

Consistent Commenting

Adding comments where necessary to explain complex logic or important details. This helps future developers understand the context and reasoning behind certain code decisions.

Code Formatting

Following rules around indentation, spacing, and line length to ensure uniformity across files, which reduces cognitive load when navigating code.

Introduction to TDD & BDD

  • TDD: Write tests first, build code to pass tests, then refactor.
  • BDD: Emphasizes behavior with scenarios and collaboration.

TDD Process

A simple cycle as Red → Green → Refactor

  • Red: Write a test for a new feature. Initially, this test will fail.
  • Green: Write just enough code to make the test pass.
  • Refactor: Clean up the code, making it more efficient or easier to maintain.

Counter Component (Goal State)

Write the Test (Red State)

Write Just Enough Code to Pass the Test (Green State)

Refactor (Refactor State)

After refactoring done, we re-run the tests to ensure everything still works correctly.

BDD Process

BDD focuses on behavior using scenarios often written in Gherkin syntax:

  • Given some context
  • When an action is taken
  • Then a specific outcome is expected
  • Adds additional conditions or details to the scenario

Gherkin Syntax

Gherkin is the language used to write BDD scenarios in a structured format that is easy to read and understand by all team members.

  • Given Given some context or setup
  • When When an action is taken or event triggered
  • Then Then a specific outcome is expected
  • And/But Adds additional conditions or details to the scenario

Best Practices for TDD & BDD

  • Clear, concise test cases
  • Tests should be independent
  • Descriptive names for tests
  • Refactor tests regularly

Web Performance Optimization

Strategies to enhance loading times and user experience:

  • Minify CSS, JS, and HTML
  • Lazy load images and scripts
  • Use efficient caching strategies
  • Optimize asset delivery with CDNs

Minifying Assets

  • Minification is the process of removing unnecessary characters (like spaces, comments, and line breaks) from code files (JavaScript, CSS, and HTML) to reduce their size without changing the functionality.
  • Minifying assets reduces the amount of data that needs to be transferred over the network. Smaller file sizes lead to faster download times, improving the page load speed.
  • This is particularly important for mobile users, where bandwidth can be limited.

Example:

Content Delivery Networks(CDN)

  • A CDN is a network of distributed servers that store copies of your website's static assets (such as images, JavaScript, and CSS files) in multiple geographic locations. When a user requests a resource, the CDN serves it from the server closest to the user, reducing the latency and speed of content delivery.
  • By distributing assets to edge servers around the world, the CDN minimizes the physical distance between users and content. This reduces latency and improves load times, especially for users located far from your primary server.
  • CDNs offload traffic from the origin server, helping to handle spikes in demand without overwhelming the server. This improves the overall scalability of your website.
